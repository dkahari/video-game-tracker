# Data Model

This document defines the initial data structures for the Video Game Tracker MVP. It is the reference point for forms, storage, filtering, and TypeScript types.

## Design Goals

- Keep the model small enough for an MVP
- Support the planned list, detail, dashboard, and form screens
- Make storage and validation straightforward
- Leave room for future expansion without breaking everything

## Core Types

Suggested TypeScript shapes for the initial app:

```ts
export type GameStatus = 'backlog' | 'playing' | 'completed' | 'dropped'

export interface Game {
  id: string
  title: string
  platforms: string[]
  status: GameStatus
  purchaseDate: string | null
  hoursPlayed: number
  completionPercent: number
  genreTags: string[]
  notes: string
  createdAt: string
  updatedAt: string
}

export interface GameLibraryStorage {
  version: number
  games: Game[]
  updatedAt: string
}
```

## Field Definitions

### `Game`

- `id`
  - Type: `string`
  - Purpose: unique identifier for a game record
  - Notes: should be generated by the app

- `title`
  - Type: `string`
  - Required: yes
  - Purpose: display name of the game

- `platforms`
  - Type: `string[]`
  - Required: yes
  - Purpose: the platforms the user owns or plays the game on
  - Notes: array allows multi-platform titles

- `status`
  - Type: `GameStatus`
  - Required: yes
  - Purpose: current state of the game in the user's library

- `purchaseDate`
  - Type: `string | null`
  - Required: no
  - Purpose: when the game was bought
  - Notes: store as ISO-like date string for form compatibility

- `hoursPlayed`
  - Type: `number`
  - Required: yes
  - Default: `0`
  - Purpose: rough playtime tracking

- `completionPercent`
  - Type: `number`
  - Required: yes
  - Default: `0`
  - Purpose: simple progress indicator from 0 to 100

- `genreTags`
  - Type: `string[]`
  - Required: no
  - Default: `[]`
  - Purpose: optional lightweight categorization

- `notes`
  - Type: `string`
  - Required: no
  - Default: `''`
  - Purpose: freeform comments about the game

- `createdAt`
  - Type: `string`
  - Required: yes
  - Purpose: record creation timestamp

- `updatedAt`
  - Type: `string`
  - Required: yes
  - Purpose: last update timestamp

### `GameLibraryStorage`

- `version`
  - Type: `number`
  - Purpose: allow future storage migrations

- `games`
  - Type: `Game[]`
  - Purpose: full library contents

- `updatedAt`
  - Type: `string`
  - Purpose: timestamp of the last library-level change

## Status Values

- `backlog`
  - Owned or added, but not currently being played

- `playing`
  - Actively being played

- `completed`
  - Finished to the user's satisfaction

- `dropped`
  - Intentionally stopped and not actively pursuing

## Validation Rules

Initial validation rules for the app:

- `title` must not be empty
- `platforms` must contain at least one value
- `status` must be one of the allowed status values
- `hoursPlayed` must be `0` or greater
- `completionPercent` must be between `0` and `100`
- `purchaseDate` can be empty, but if present should be a valid date string

## Form Defaults

Suggested defaults when creating a new game:

```ts
const defaultNewGame = {
  title: '',
  platforms: [],
  status: 'backlog',
  purchaseDate: null,
  hoursPlayed: 0,
  completionPercent: 0,
  genreTags: [],
  notes: '',
}
```

## Filtering And Display Implications

The model should support:

- filtering by `status`
- filtering by platform
- searching by `title`
- dashboard counts by status
- recent activity based on `updatedAt`

## Future Expansion Ideas

These are intentionally not part of the MVP model yet:

- cover image URL
- developer or publisher
- release date
- rating
- favorite flag
- completion date
- play sessions
- wishlist state

## Open Questions

- Whether `platforms` should remain freeform strings or move to a controlled list later
- Whether `completed` should eventually include a separate `completedAt` field
- Whether `purchaseDate` should later become part of a broader ownership model
