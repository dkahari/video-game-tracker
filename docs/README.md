# Docs Index

This folder holds the project's working reference docs.

Use this file as the quick "start here" guide for what each document is for and when it should be updated.

## Core References

- `architecture.md`
  - Purpose: current architectural direction, technical boundaries, and default implementation shape
  - Update when: a task changes architecture, storage strategy, route strategy, context boundaries, or major technical direction

- `project-brief.md`
  - Purpose: concise product summary, goals, audience, and scope
  - Update when: the product framing or MVP goals change

- `sitemap.md`
  - Purpose: planned screens, routes, and major user flows
  - Update when: screens, route structure, or navigation flows change

- `data-model.md`
  - Purpose: current domain data shape and field definitions for the MVP
  - Update when: fields, statuses, or storage shape change

## Project Tracking

- `ticket-tracker.md`
  - Purpose: lightweight in-repo project memory, current work, next steps, and handoff notes
  - Update when: a session ends, a task changes status, or the next recommended step changes

- `decision-log.md`
  - Purpose: durable technical decisions and why they were made
  - Update when: a decision is important enough that future sessions should not rediscover it from scratch

## Workflow Notes

- `frame0-workflow.md`
  - Purpose: practical workflow for generating Frame0 wireframes while keeping pages editable
  - Update when: the Frame0 process changes, the MCP workaround changes, or a better wireframing pattern is found

## How To Use This Folder

- Read `architecture.md`, `ticket-tracker.md`, and `decision-log.md` first when re-entering the project.
- Use `ticket-tracker.md` for short-lived session memory and handoff context.
- Use `decision-log.md` for durable choices that should remain true across sessions.
- Use workflow-specific docs like `frame0-workflow.md` when a task depends on a known process.

## Update Rule

If a task changes how the project is built, how the product is shaped, or how a repeated workflow should be handled, update the relevant doc in this folder during the same task.
